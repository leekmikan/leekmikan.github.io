Github pages上でSEO対策

・Github pages上でSEO対策を箇条書きでまとめたもの

・多数のサイトの情報をまとめた

・PageSpeed Insightsでの得点の増加を優先

・Github pagesでの制約(サーバーの設定ができないなど)以外での対策

・もしかしたらもっと良い方法があるかも...

1.metaタグを付ける(多分必須)

次のmetaタグをつけないと、Google側がクソページ扱いしてきてインデックス登録をしてくれない
→検索画面にでてこない


<meta content='width=device-width,initial-scale=1.0,minimum-scale=1.0' name='viewport'/>

2.scriptにdeferを付ける

ウェブページの読み込み時間増加による遅延を軽減

JavaScriptが実行できているか確認すること


<script defer src="aaa.js"></script>

3.cssを遅延読み込みする

media="print" onload="this.media='all'"　を付け加える

ウェブページの読み込み時間増加による遅延を軽減

cssが読み込めているか確認すること


<link rel="stylesheet" href="des.css" media="print" onload="this.media='all'">

4.読み込み完了後にbodyを表示する

「3.cssを遅延読み込みする」だけだとレイアウトシフトが発生して、モバイル端末の得点が高いのにPCの得点が低いという状況になるので、次の手順を踏む

1.適当なJavaScriptに次のコードを書く　すでにwindow.onloadがあればその中に書く

今回はこれをinit.jsとする


window.onload = function(){
   document.body.style.display = "block";
}

2.body部分を書き換えて、非表示に設定する


<body style="display: none;">

3.　1で作ったファイルを読み込む


<script defer src="init.js"></script>

5.その他

Github pages関係なく、共通の対策

多分PageSpeed Insightsで表示されるので分かるはず

今思いつく範囲で箇条書き

  1. 画像サイズを小さくする(100KB以下だと良さそう?)　ただし、解像度が悪すぎると若干減点される
  2. imgタグ内にalt="画像の説明"を書く
  3. inputはlabelタグで囲む/labelタグにfor="input要素のname名"を書いて使う
  4. <meta name="description" content="このページの説明">を書く

最後に.テンプレHTMLファイル

HTML部分のテンプレート

アイコン用のlinkタグ付き


<!DOCTYPE html>
<html lang="ja">
	<head>
		<meta charset="uft-8">
		<title>タイトル</title>
		<meta content='width=device-width,initial-scale=1.0,minimum-scale=1.0' name='viewport'/>
		<meta name="description" content="このページの説明">
		<link rel="stylesheet" href="cssファイル.css" media="print" onload="this.media='all'">
		<link rel="icon" type="image/jpg" href="アイコン.jpg">
		<link rel="apple-touch-icon" sizes="180x180" href="アイコン.jpg">
		<iframe src="../google.html" style="display: none;"></iframe>
		<script defer src="「4.読み込み完了後にbodyを表示する」で作ったJSファイル.js"></script>
		<body style="display: none;">
			<!--本文-->
		</body>
	</head>
</html>
